Task 3: CCD Inverse Kinematics

Implementation-oriented draft for cyclicCoordinateDescentMethod(...) in exercise2/js/kinematic-linkage.js.

1) Goal

Find new DOF values \(\theta\) such that the end-effector position gets close to the target position. CCD does this iteratively: update one joint at a time, usually from tip to base.

2) Core Idea of One Joint Update

For current joint \(j\), define in world coordinates:

  • Joint position: \(\mathbf{p}_j\)
  • Current end-effector position: \(\mathbf{p}_e\)
  • Target position: \(\mathbf{p}_t\)

Build vectors from the joint to end-effector/target:

\[ \mathbf{u} = \mathbf{p}_e - \mathbf{p}_j, \quad \mathbf{v} = \mathbf{p}_t - \mathbf{p}_j \]

For an unconstrained 3D joint, the alignment angle could be computed from:

\[ \cos\Delta\theta = \frac{\mathbf{u}\cdot\mathbf{v}}{\|\mathbf{u}\|\,\|\mathbf{v}\|} \]

Clamp the cosine argument to \([-1, 1]\) to avoid numerical issues.

3) Axis Constraint for This Exercise

In this task, each joint is revolute with exactly one allowed local axis (X, Y, or Z). So we do not apply arbitrary 3D rotations. We solve in the plane perpendicular to that axis.

  1. Get current joint rotation matrix \(R_j\) (world orientation of joint frame).
  2. Convert vectors to local coordinates: \[ \mathbf{u}_{loc} = R_j^T \mathbf{u}, \quad \mathbf{v}_{loc} = R_j^T \mathbf{v} \]
  3. Choose 2D components based on allowed axis:
    • axis X: use \((y,z)\)
    • axis Y: use \((x,z)\)
    • axis Z: use \((x,y)\)
  4. Compute signed 2D angle \(\Delta\theta_j\) and update only that DOF: \[ \theta_j \leftarrow \theta_j + \Delta\theta_j \]
Why this works: the local axis constraint is enforced naturally because only the planar angle around that axis is changed.

4) Full CCD Loop (Tip to Base)

  1. Start from current dofValues (copy to outputValues).
  2. Repeat for numIterations:
  3. Evaluate current FK (pose) with outputValues.
  4. Check end-effector error: \[ e = \|\mathbf{p}_t - \mathbf{p}_e\| \] If \(e < \varepsilon\), stop.
  5. For joints from last to first:
    • Compute \(\mathbf{u}, \mathbf{v}\).
    • Compute constrained \(\Delta\theta_j\).
    • Update only this joint DOF.
    • Recompute FK before processing the next parent joint.

5) Mapping to Your Existing Fields

  • this.dofNames[j]: gives the DOF variable to update.
  • this.rotationAxes[j]: gives local axis constraint (X, Y, Z).
  • this.localCoordinateFrames: symbolic transforms; evaluate with current DOFs to get current world joint/end-effector positions.
  • this.forward(outputValues): update scene objects after optimization step or at the end.

6) Stability Rules (Important)

  • If \(\|\mathbf{u}\|\) or \(\|\mathbf{v}\|\) is near zero, skip that joint update.
  • Clamp acos input to \([-1,1]\).
  • Optionally clamp \(|\Delta\theta_j|\) (for smoother behavior and less oscillation).
  • Always keep a max iteration count for unreachable targets.

7) Minimal Mathematical Summary

\[ \mathbf{u} = \mathbf{p}_e - \mathbf{p}_j,\quad \mathbf{v} = \mathbf{p}_t - \mathbf{p}_j,\quad \theta_j \leftarrow \theta_j + \Delta\theta_j \]

with \(\Delta\theta_j\) computed in the local 2D plane orthogonal to the joint's fixed axis.
